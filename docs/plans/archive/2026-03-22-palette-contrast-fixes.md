## Cross-Cutting Patterns (READ FIRST)

Every file has some combination of these 5 failure patterns. Understanding them prevents redundant explanation per task.

### Pattern A: rgba opacity too low on dark backgrounds
Dark-theme files use `rgba(R,G,B, opacity)` for text hierarchy. The opacity values are set too low for WCAG AA (4.5:1 for normal text, 3:1 for large text ≥18px or ≥14px bold).

**Fix:** Bump opacity values. The exact bump depends on the base color and background — provided per file.

### Pattern B: Accent color in luminance dead zone
Some accent colors (~40-60% relative luminance) fail as BOTH text-on-light-bg AND as button-bg-with-white-text. They're too dark for white text on top, too light to read on white/cream backgrounds.

**Fix:** Split into two variants — a darker version for text/small UI on light backgrounds, keep original for large decorative use. OR darken the single accent enough to pass as button bg with white text (usually sufficient since most usage is buttons).

### Pattern C: Muted text color fails on one bg type
A single muted color is used on both light and dark backgrounds but only passes on one. Common in files that alternate light/dark sections.

**Fix:** Add a contextual override (e.g., `.bg-d .mu{color:...}`) for the failing background type.

### Pattern D: Footer bottom text near-invisible
Every file uses very low opacity or very dark hex for the copyright/discount line. This is the single most common failure.

**Fix:** Bump to minimum viable contrast. This text is intentionally subdued but still needs to be readable.

### Pattern E: Highlighted level card text
The Level 1 "Start Here" card uses the accent color as background. Text on it (especially the `.pbox` label "Included in your") often fails because it uses reduced opacity.

**Fix:** Increase text opacity on highlighted card, or switch text color approach.
